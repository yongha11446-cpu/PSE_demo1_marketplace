# PSE_demo1 — 아스펜 공정모사 연구 비서 플러그인

리그노셀룰로오스 → SAF(ATJ) 같은 화학공정 모사 연구를, 문헌탐색부터 노벨티 도출·가상 시뮬레이션·연구노트 기록까지 한 번에 돕는 Claude Code 플러그인.

## 들어 있는 것

| 종류 | 이름 | 설명 |
|---|---|---|
| 커맨드 | `/초기화` | 새 프로젝트 세팅 — 템플릿 CLAUDE.md·README.md 복사 + 폴더 골격 생성 |
| 커맨드 | `/주제분석` | 주제 유형 분류(차별화/직렬연결/신규방법)·논문탐색·노벨티·연구방향 공정체인 3대안 |
| 커맨드 | `/분석시작` | 수집 논문 전체 정독 → 한계점 분류 → 노벨티 점수화 |
| 커맨드 | `/가상시뮬레이션` | 대안별 블록 수지·수렴·물리타당성 검증 + 본모사 후 대조 |
| 커맨드 | `/업데이트` | 인자없음=바뀐 파트 전체 자동 갱신(all), **프로젝트 연구노트.md만** anti-stale 기록 (글로벌 미반영) |
| 커맨드 | `/글로벌업데이트` | 재사용 가능한 문헌 지식만 글로벌 마스터 연구노트에 반영 — **사용자가 명시 호출할 때만** |
| 에이전트 | `논문탐색가` `정독분석가` `노벨티설계가` | 대량 검색·정독·노벨티를 병렬로 분리 |
| 훅 | 갱신 리마인드 | 아스펜 스트림 붙여넣기 감지 → 기록 프로토콜 자동 안내 |
| 헬퍼 | `make_refs` `make_lit_excel` `make_novelty_excel` `make_vsim_excel` | 엑셀·refs 자동 생성 (Python) |

## 요구사항

- **OS**: Windows (훅이 PowerShell 기반). Mac/Linux는 훅 일부가 동작하지 않음.
- **Python** + `openpyxl` — 엑셀 헬퍼용. 설치:
  ```
  pip install openpyxl
  ```

## 처음 한 번: 새 프로젝트 세팅

새 연구 폴더에서 **`/초기화`** 한 번이면 끝난다:
- `templates/CLAUDE.md` → 프로젝트 루트 `CLAUDE.md` (한국어·표·단위 규칙, 검증/발췌/anti-stale 3원칙, 블록 설계 협업 루프)
- `templates/README.md` → 프로젝트 루트 `README.md` (명령 6종·서브에이전트·핵심 습관·폴더 규칙을 담은 프로젝트용 사용 가이드)
- 권장 폴더 골격 생성: `data/raw` · `data/results` · `refs` · `scripts` · `sections`
- 기존 파일은 덮어쓰지 않는다(보호). 새로 깔려면 `/초기화 덮어쓰기`.

그 뒤 `CLAUDE.md`의 `<...>`(주제·목표·현재 단계)만 채우고 `/주제분석 <주제>`로 시작한다.

- **연구노트 2층 구조**:
  - `/업데이트` → **이 프로젝트의 `연구노트.md`만** 갱신한다(항상). 글로벌은 절대 건드리지 않는다.
  - `/글로벌업데이트` → **사용자가 명시 호출할 때만** 글로벌 마스터(`~/.claude/연구_라이브러리/마스터_연구노트.md`)에 반영한다. 근거(저자·Table) 있고 프로젝트 횡단 재사용되는 문헌 지식만 게이팅 통과.
  - 이유: 글로벌은 모든 프로젝트가 매 세션 읽는 공용 색인이라, 프로젝트 한정 값(투입량·조성·방침)이 섞이면 다른 프로젝트를 오염시킨다.
- **글로벌 연구노트** 파일은 `/글로벌업데이트` 첫 실행 시 자동 생성된다.
  미리 만들려면 `templates/마스터_연구노트.md`를 `~/.claude/연구_라이브러리/` 로 복사한다.

## 경로 규칙 (개발자용)

- 번들 스크립트는 `${CLAUDE_PLUGIN_ROOT}/scripts/...` 로 참조한다(플러그인 위치 자동 해석).
- 글로벌 연구노트는 `~/.claude/연구_라이브러리/...` (사용자 홈) 에 누적한다.
- 절대경로(`C:\Users\...`)는 쓰지 않는다.

## 참고

- 이 플러그인은 **연구 도구 전용**이다. 위험명령 확인(guard-ask)·완료 알림음 같은
  범용 훅은 포함하지 않는다(개인 `~/.claude/settings.json`에서 따로 설정).
- 모든 커맨드/에이전트는 **추측 금지**(근거 없는 값은 `확인 필요`) 원칙으로 동작한다.
