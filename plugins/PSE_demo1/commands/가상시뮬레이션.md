대상: **$ARGUMENTS**

아래 "가상 시뮬레이션 한방" 파이프라인을 수행한다. 결과는 한국어, 표 중심, **단위 항상 명시**(°C, bar, atm, wt%, kg/hr, kmol/hr).
**추측 금지** — 확인 안 된 조건은 절대 지어내지 말고 `확인 필요`로 표기한다. 모든 수치엔 근거(논문번호+이름, 저자·연도·Table/쪽)를 병기한다.

> 목적: 아스펜 본모사 전에, 각 대안이 **물리적으로 말이 되는지(타당성)** 와 **수지가 닫히고 수렴하는지** 를 가벼운 계산으로 먼저 검증한다. 보고서가 아니라 본모사 진입 가능 여부 판정이 목적.

## 0. 대안 정의·전제 확인
- `연구노트.md`(없으면 `근거/연구노트.md`) + `data/results/주제분석_*.xlsx`(연구방향 시트)에서 **대안 1~3 정의**를 먼저 읽는다(없으면 사용자에게 확인).
- 각 대안 = 공정 블록 체인 + 블록별 기준논문(번호+이름). 빠진 조건(촉매·압력·분리 등)은 **한 줄로 물어본 뒤** 진행하거나 `확인 필요`.
- 탐색/계산 기준(재순환 포함 여부, Rstoic vs RYield, 전환율 출처 등)이 안 주어졌으면 한 줄로 확인.

## 1. 공정 블록 분해 (대안마다)
각 대안을 **단위 블록 체인**으로 편다. 표준 골격(ATJ):
`전처리·가수분해·발효 → 에탄올 정제 → 탈수(→에틸렌) → 올리고머화 → 수소화 → 분리·증류(제트유 컷)`
- 블록마다: **기준논문(번호+이름)** · 온도(°C) · 압력(bar) · 전환율/수율 · 선택도 · 주요 화학량론 · 상(phase).
- 블록 단위로 한다(CLAUDE.md "블록 단위 검증"): 한 블록을 수지+문헌검증으로 통과시킨 뒤 다음 블록.

## 2. 근거 발췌 — 서브에이전트 병렬
블록별로 `논문탐색가`를 띄워 **원본을 직접 읽고** 조건을 발췌한다(한 메시지에 여러 Task 병렬).
- `refs/NN_*.pdf`가 있으면 **PDF 정독**, 없으면(`[업로드필요]`) **web에서 원문/출판본을 확인**해 발췌. 둘 다 안 되면 `확인 필요` + 정확한 제목으로 업로드 요청.
- 발췌 형식: 값 + (저자, 연도, Table/Fig, 쪽). md를 그대로 믿지 말고 원문 대조.

## 3. 가상 시뮬레이션 계산 — scripts/ 파이썬
무거운 계산은 전부 `scripts/가상시뮬_<대안>.py`로 뺀다(대화창엔 결과 표만).
- 기준 처리량(kg/hr 또는 kmol/hr)에서 시작 → 블록별 **물질수지**(화학량론·전환율 적용) → 주요 블록 **에너지수지**(필요시) → 재순환 있으면 **수렴 반복**.
- 각 줄 한국어 주석, 초보자 기준. 입력 스트림은 `data/raw/`(수정 금지)에서, 결과는 `data/results/`로.

## 4. 수렴·물리적 타당성 판정
블록별·전체에 대해:
- **수지 닫힘오차**(%) — 입력 대비 출력 (목표 < 0.1% 또는 기준 명시).
- **재순환 수렴** — 반복 횟수·수렴 여부(tear stream 기준값).
- **물리적 타당성** — 온도/압력/상이 촉매·반응 조건과 모순 없는지, 음수·100% 초과 전환 등 경고.
- 판정: 대안별 `수렴 OK / 경고 / 실패(사유)`.

## 5. 엑셀 정리
대안별 결과를 엑셀로. JSON을 만들어 헬퍼 실행:
`python "${CLAUDE_PLUGIN_ROOT}/scripts/make_vsim_excel.py" <입력JSON> <출력xlsx>`
- 출력: `data/results/가상시뮬_<주제요약>.xlsx`.
- 시트 구성(대안마다 1시트 + 요약 1시트). 열: `블록 | 기준논문(번호+이름) | 온도 | 압력 | 전환율/수율 | 선택도 | 주요 스트림(kg/hr) | 수렴/닫힘오차 | 근거(저자·연도·Table/쪽)`.
- 파일이 열려 있어 저장 실패 시 `_v2` 새 이름으로 저장 후 알림.

## 6. 마무리 — 기록·스킬화
- 확정 조건·근거는 **`/업데이트 <블록>`**(예: `/업데이트 탈수`)으로 연구노트(`연구노트.md`)에 기록한다(같은 항목 덮어쓰기·새 항목 추가, 글로벌 `~/.claude/연구_라이브러리/마스터_연구노트.md`에도 반영).
- 아스펜 오류·해결이 나오면 `~/.claude/연구_라이브러리/오류해결대장.md`에 누적.
- **대안 1이 사용자의 실제 연구와 일치**하면, 이 파이프라인(블록 체인+조건+계산)을 재사용 가능한 **스킬**로 만들 것을 제안한다.
