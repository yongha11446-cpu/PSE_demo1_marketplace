대상: **$ARGUMENTS**

이 명령은 **사용자가 명시적으로 호출할 때만** 글로벌 마스터 연구노트(`~/.claude/연구_라이브러리/마스터_연구노트.md`)에 항목을 반영한다. `/업데이트`는 글로벌을 절대 건드리지 않으므로, 글로벌 반영은 항상 이 명령으로만 한다. 결과는 한국어. **추측 금지** — 근거(저자·연도·Table/쪽) 없는 값은 반영하지 않는다.

> **왜 분리하나**: 글로벌 마스터는 모든 프로젝트가 매 세션 읽는 공용 색인이다. 프로젝트마다 주제가 다르므로, 특정 프로젝트에만 의미 있는 값까지 글로벌에 넣으면 다른 프로젝트에서 무관한 정보를 읽게 되어 오염된다. 그래서 글로벌엔 **프로젝트를 바꿔도 재사용되는 문헌 지식만** 올린다.

## 1. 게이팅 — 무엇이 글로벌에 올라가도 되는가
각 후보 항목을 아래 기준으로 판정한다. **둘 다 만족해야** 글로벌 반영 대상이다.

| 글로벌 반영 ✅ (둘 다 충족) | 글로벌 금지 ❌ (하나라도 해당) |
| --- | --- |
| ① 근거가 있다 — 저자·연도·Table/쪽 또는 공인 출처 | 사용자 지정 임의값·특정 feedstock 투입량·스트림 수치 |
| ② 재사용된다 — 다른 프로젝트에서 "그거 어떻게 했더라" 검색해 그대로/응용 적용 가능 | 작업 방침·진행상황·중간 계산결과·이 프로젝트 한정 결정 |
| 예: NREL 전처리 전환율표, 촉매 선택도/전환율, 분리 회수율 표준 | 예: "Kenaf 3745.83 kg 투입", "조성 56.31/27.02/16.67", "독립재현 방침" |

- `$ARGUMENTS`가 비어 있으면: 대화/연구노트에서 **글로벌 후보**(위 ✅ 기준 통과)를 골라 목록으로 제시하고, **어떤 항목을 올릴지 사용자에게 확인**한 뒤 진행한다. 임의로 전부 올리지 않는다.
- 후보가 ❌ 기준에 걸리면 그 이유를 한 줄로 말하고 **반영하지 않는다**.

## 2. 마스터 찾기/만들기
- `~/.claude/연구_라이브러리/마스터_연구노트.md`를 연다. 폴더/파일이 없으면 만든다(아래 골격).
  ```
  # 마스터 연구노트 (프로젝트 횡단)

  > 모든 연구 프로젝트의 **재사용 가능한 문헌 지식**만 한 줄씩 누적하는 검색용 색인.
  > `/글로벌업데이트`로만 갱신(사용자 명시 호출). `/업데이트`는 글로벌을 건드리지 않는다.
  > 같은 프로젝트+파트+항목은 덮어쓰기, 새 항목은 추가.

  | 날짜 | 프로젝트 | 단위공정/주제 | 무엇을 결정 | 근거(저자, 연도, Table/쪽) | 어떻게 적용 | 파일 위치 |
  |---|---|---|---|---|---|---|
  ```

## 3. 반영 (덮어쓰기/추가)
- 평탄형 표 한 줄로 적는다: `| 날짜 | 프로젝트 | 단위공정/주제 | 무엇을 결정 | 근거 | 어떻게 적용 | 파일 위치 |`.
- **같은 프로젝트+파트+항목이면 그 줄을 덮어쓴다**(최신만 유지), 없으면 추가한다.
- 날짜는 오늘(YYYY-MM-DD 또는 기존 표 형식에 맞춘다). 프로젝트명은 현재 폴더명/연구노트 제목에서 가져온다.

## 4. 요약 보고
- 올린 항목(✅)과 게이팅에서 걸러진 항목(❌+이유)을 표로 구분해 보고한다. 파일 경로(마스터)도 함께.
