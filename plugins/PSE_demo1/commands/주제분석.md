연구 주제: **$ARGUMENTS**

아래 "주제 분석 한방" 파이프라인을 수행한다. 결과는 한국어, 표 중심. 추측 금지(확인 안 된 값은 `확인 필요`).

## 권장 폴더 구조 (새 프로젝트면 먼저 안내)
이 파이프라인(`/주제분석`→`/분석시작`→`/가상시뮬레이션`→`/업데이트`)은 아래 구조로 산출물을 쌓는다.
현재 폴더에 없으면 **사용자에게 한 줄로 안내**하고(자동 생성은 각 단계가 알아서 함), 있으면 그대로 쓴다.
```
프로젝트/
├─ data/
│  ├─ raw/        원자료(수정 금지) — 입력 스트림 등
│  └─ results/    엑셀 산출물 (주제분석/분석시작/가상시뮬)
├─ refs/          논문 PDF·00_문헌목록.md (/주제분석이 채움)
├─ scripts/       계산 스크립트 (/가상시뮬레이션이 만듦)
├─ sections/      공정별 스트림·운전조건 문서
└─ 연구노트.md    설계 결정 색인 (/업데이트가 관리)
```
- 이 단계(`/주제분석`)는 `refs/`와 `data/results/`를 만든다. 나머지는 후속 단계에서 생긴다.

## 0. 주제 유형 자동 분류 + 탐색 기준
먼저 이 주제가 아래 **3유형 중 무엇인지 자동 판단**하고, **근거 1줄과 함께 제시해 사용자에게 확인/수정받은 뒤** 진행한다(추측 금지 — 단정하지 말고 "이 주제는 B(신규 조합)로 보입니다, 맞나요?"처럼 1줄 확인).

- **A. 선행연구 풍부형 (차별화 필수)** — 제품/공정이 이미 많이 연구됨(예: 폐타이어 열분해유→Jet). → 기존 SOTA·**반복되는 한계**를 모아 **명시적 차별점**(새 촉매·조건·통합·원료)을 노벨티로 잡는다. 검색 기준: 최신 SOTA + 한계 명시 논문 우선.
- **B. 신규 조합형 (직렬 연결)** — 목표 공정이 통째로는 세상에 없음(예: Kenaf→SAF). → 목표 제품을 **중간체 단계로 분해**(원료→중간체→…→제품), **링크마다 기준논문 1편**을 찾아 **링크 N의 출력 스트림 = 링크 N+1의 입력**으로 잇는다. 검색 기준: 링크별 최적 논문(예: 바이오매스→에탄올 논문 + 에탄올→SAF 논문).
- **C. 기존제품·신규방법형** — 제품은 있으나 방법이 신규(예: green H2+CO2→e-메탄올을 kinetic 반응기로). → **방법 논문(kinetic·신규촉매 등) + 베이스라인 공정 논문**을 가져온다. 방법 논문은 외부 계산(MATLAB 등)으로 조건을 산출해야 할 수 있음을 명시. 노벨티 = 그 방법의 적용.

탐색 기준(예: Kinetic 제외 / Rstoic 우선 / 전환율 숫자 명시·그래프 금지 / 특정 생성물): 사용자가 줬으면 위 유형 기준에 **추가로 합치고**, 안 줬으면 **한 줄로 물어본 뒤** 진행.

## 1. 논문 분석팀 — 서브에이전트 병렬 (넓게 + 깊게 동시)
주제를 **3~6개 하위영역으로 쪼개** 각 영역에 `논문탐색가`를 **하나씩, 한 메시지에 동시 상한까지 병렬로** 띄운다(같은 에이전트 종류를 여러 개 동시에 — 정의 복제 불필요). 각 팀원은 자기 영역을 **깊게** 보고(주제 적합도·최신성·저널등급·인용수·공정 입출력), 합치면 자동으로 **넓게**(합산 20~100편) 된다.
- **분할 기준 — 0단계에서 정한 유형을 따른다**:
  · **B(신규 조합)**: **공정 링크별**로 배분 — 원료→중간체→…→제품의 각 링크에 팀원 1명(예: (바이오매스→에탄올) / (에탄올→에틸렌→올리고머→SAF)). 링크별 최적 기준논문을 찾는 게 목표.
  · **A(선행연구 풍부)**: **SOTA + 한계 각도** — 최신 대표공정 / 반복되는 한계 / 차별화 시도(새 촉매·조건·통합) / TEA·LCA / 리뷰.
  · **C(기존제품·신규방법)**: **방법 + 베이스라인** — 방법 논문(kinetic·신규촉매) / 기존 베이스라인 공정 / 조건계산(MATLAB 등) / 검증·비교.
  · 유형이 애매하면 일반 분할 — 다단계면 공정 단계별, 단일이면 각도별(촉매 / 공정모사·운전조건 / TEA / LCA / 리뷰).
- 각 팀원에게 **자기 논문 공정의 입력(feed)·출력(product) 스트림과 중간체를 함께 발췌**하라고 지시한다(저자·Table·쪽 병기). → 3단계에서 논문을 출력→입력으로 직렬 연결할 때 쓴다.
- 각 팀원: 자기 영역에서 후보를 충분히 수집(합산이 목표 편수에 닿게) → 상위를 깊게 평가. 못 여는 논문은 `접근불가` 표시.
- 메인은 팀 결과를 **병합**: 중복 제거 → 번호 통합 → 전역 **학습 2편(⭐)** 선정 → 동향·3대안 종합.
- 대량 검색·정독을 팀에 떼어 **메인 대화는 깨끗이** 유지한다.

## 2. 랭킹 + 필독 선정
- 평가된 문헌에 **종합점수**를 매긴다 = **주제 적합도(1순위, 연관 선행연구 많은 주제는 절하) + 최신성(2순위) + 저널등급/IF(3순위) + 인용수(보조)**의 가중 조합(논문탐색가가 산출). 종합점수 내림차순으로 **순위**를 부여한다.
- **별표 논문 선정 기준 = 위 주제 적합도(1순위)를 따른다** — ⭐ 학습 2편과 📖 필독 N편 **둘 다** 주제 적합도가 높은(그리고 선행 포화가 아닌) 논문을 우선 선정한다.
- **필독(📖) 선정 — 당신이 직접 출력해 읽을 배경지식용**: 학습 2편(⭐, 아래 유지)과 **별도로**, 주제 적합도 기준 종합점수 **상위 N편**을 필독으로 지정한다. **N은 기본 5편**, `$ARGUMENTS`에 "N편"이 주어지면 그 수.
  - 📖 필독은 **사용자가 출력해 읽어 배경지식을 잡는 용도**이며, 다음 단계의 자동 분석 대상과는 무관하다(다음 단계 `/분석시작`은 수집한 **전체** 논문을 분석한다). 따라서 PDF 업로드를 분석의 전제로 강요하지 않는다.

## 3. 종합 출력 (대화창)
- **학습 논문 2편(⭐)**: ① 연구 경향성 대표 1편 + ② 주제 자체 대표 1편 (제목·저자·연도·한 줄 이유) + 연구 동향 3~5줄.
- **문헌 가치평가표**(종합점수 내림차순): 순위 | 종합점수 | 번호 | 공정 단계(있으면) | 제목 | 저자(연도) | 적합도(선행 유무) | 최신성(연도) | 저널(등급/IF) | 인용수 | feed→product(중간체) | 접근성 | 필독(📖)
- **필독 N편 목록**: 종합점수 상위 N편을 📖로 따로 모아 "이건 직접 출력해 읽으며 배경지식을 잡으세요"라고 안내.
- **연구방향 3대안 (공정 체인으로)**: 최선 1 + 대안 2. 각 대안을 **출력→입력으로 이어진 공정 체인**으로 제시한다(조합 문헌 번호만 나열하지 말 것):
  - **공정 체인**: `원료 → [링크1 기준논문 번호+이름] → 중간체 → [링크2 …] → … → 제품` (B형은 필수, A·C형도 핵심 단계를 체인으로). 각 링크에 출력 스트림이 다음 링크 입력과 맞는지 한 줄로 명시.
  - **노벨티**: 유형에 맞게 — A=기존 대비 차별점 / B=직렬 연결(통합) 자체 + 링크별 개선 / C=신규 방법 적용.
  - **무엇을 · 왜 · 리스크**도 함께.

## 4. refs 자동 구축 (번호·링크·안내)  ← 엑셀보다 먼저
수집한 전체 문헌에 **번호를 부여**하고 `refs/`에 정리한다(엑셀 번호와 동일).
- papers 목록을 JSON으로 만들어(`rank, short, name, journal, access, url, study, mustread`) 헬퍼 실행:
  `python "${CLAUDE_PLUGIN_ROOT}/scripts/make_refs.py" <papers.json> <프로젝트>/refs`
- 결과: `refs/00_문헌목록.md`(번호↔논문, ⭐학습, 📖필독, 상태) + 없는 논문은 `NN_약칭_[업로드필요].txt`(링크+넣는 법) 자동 생성. 이미 `NN_*.pdf`가 있으면 ✅로 처리.
- 학습 2편은 `study:true`로 ⭐ 표시. **필독 N편은 `mustread:true`로 📖 표시**(사용자가 직접 출력해 읽을 배경지식용 — 자동 분석 대상과 무관).

## 5. 엑셀 정리
- papers, directions를 JSON으로 만들어: `python "${CLAUDE_PLUGIN_ROOT}/scripts/make_lit_excel.py" <입력JSON> <출력xlsx>`
- 출력: `data/results/주제분석_<주제요약>.xlsx` (문헌평가 / 연구방향 2시트, 번호·조합문헌 포함).
- 파일이 열려 있어 저장 실패 시 `_v2` 등 새 이름으로 저장하고 알린다.

## 마무리
- 확정된 근거(쓰기로 한 논문·조건)는 **`/업데이트 문헌`** 으로 연구노트(`연구노트.md`)에 기록한다(같은 항목 덮어쓰기·새 항목 추가, **프로젝트 `연구노트.md`만** — 글로벌 마스터는 건드리지 않는다. 재사용 가능한 문헌 지식은 `/글로벌업데이트`로만 반영).
- **다음 단계 안내**: 바로 **`/분석시작`** 을 실행하면 수집한 **전체 논문**을 정독(refs에 PDF 있으면 PDF, 없으면 web)해 한계점 분류 → 노벨티 점수화로 이어진다. (필독 📖은 그와 별개로 당신이 출력해 읽으며 배경지식을 잡는 용도.)
