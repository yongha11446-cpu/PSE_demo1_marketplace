대상: **$ARGUMENTS**

대화에서 **확정된 값·설계 결정**을 프로젝트 **연구노트**(`연구노트.md`)에 기록한다. 결과는 한국어. **추측 금지** — 확정 안 된 값은 적지 않거나 `⚠️확인필요`로 표기하고, 모든 값엔 근거(저자·연도·Table/쪽 또는 계산 출처)를 병기한다.

> **연구노트 구조 (중요 — 2층)**
> 1. `## 색인 (스캔용)` — 짧은 값만 담는 표. 한눈에 훑는 용도. 각 행은 `[▶](#dNN)`로 상세 카드에 링크.
> 2. `## 상세 기록` — 항목마다 **카드 1개**. `### <a id="dNN"></a>MM-DD · <단위공정> — <결정 요약> <상태아이콘>` 제목 + `결정/값/근거/적용/파일` 불릿.
>
> **핵심 규칙**
> - **항목(같은 대상의 결정) 단위**로 기록한다. 같은 항목이면 **색인 행과 카드를 둘 다 덮어쓴다**(옛 값 삭제, 최신만 유지 = anti-stale). 새 항목이면 색인 행과 카드를 **둘 다 추가**한다. *절대 옛 값을 그냥 밑에 덧붙이지 않는다.*
> - 색인 행과 카드는 **항상 짝**이다. 하나만 고치고 다른 하나를 빠뜨리지 않는다.
> - 상태 아이콘: ✅확정 · 📄PDF직독 · ⚠️확인필요 · 🗑폐기 · ⏳대기

## 0. 모드 분기 (ALL 기본 / 특정 파트)
`$ARGUMENTS`를 본다:

### (A) 비어 있음 또는 `all` / `전체` → **ALL 모드 (기본)**
지금까지의 **대화 전체**를 훑어 **바뀐 파트만 골라** 자동 갱신한다. 안 바뀐 파트는 절대 건드리지 않는다.
1. **후보 파트 집합**을 만든다 = 연구노트의 `## 파트 맵 (공정 체인)` 목록(있으면) ∪ `## 상세 기록`의 기존 카드 항목들 ∪ 대화에서 새로 등장한 단위공정/주제.
2. 각 후보 파트마다 대화에서 **'새로 확정된 값'이 있는지** 판정한다(확정 = 근거 있는 수치·조건·결정. 단순 언급·미확정은 제외).
3. **새 확정값이 있는 파트만** 아래 2~6단계를 적용한다(있으면 색인행+카드 덮어쓰기, 없으면 추가).
   - ⚠️ **anti-stale 가드**: 대화에서 새로 확정되지 않은 파트의 색인행·카드는 **한 글자도 바꾸지 않는다**(옛 기록 그대로 유지).
4. 여러 파트면 각각 반복 적용한다.
5. 마지막에 7단계 보고에서 **갱신된 파트(옛값→새값)** 와 **변동 없어 건너뛴 파트**를 표로 구분해 보여준다.

### (B) 특정 파트명(예: `탈수`, `노벨티`) → **타깃 모드**
그 파트 하나만 아래 2~6단계로 기록한다(기존 동작).
- 파트명이 모호하면, 대화에서 **바뀐/확정된 파트 후보**를 한 줄로 제시하고 어느 파트인지 물어본다.

## 1. 연구노트 찾기/만들기
- 프로젝트(현재 폴더)에서 연구노트를 찾는다: 우선 `연구노트.md`(루트), 없으면 `근거/연구노트.md`. **둘 다 없으면 루트에 `연구노트.md`를 새로 만든다**(아래 골격 포함).
  ```
  # 연구노트 — <프로젝트명> (이 프로젝트 설계 결정 색인)

  > 모든 수치·설계 결정을 누적한다. **변경 로그도 겸한다.**
  > `/업데이트`(인자없음=바뀐 파트 전체 자동 갱신)로 갱신, 특정 파트만은 `/업데이트 <파트>`. 같은 항목은 덮어쓰기(최신만 유지), 새 항목은 추가.

  > **사용법** — 색인표로 훑고, `상세` 링크로 점프해 근거·적용·파일을 본다.
  > 상태 아이콘: ✅확정 · 📄PDF직독 · ⚠️확인필요 · 🗑폐기 · ⏳대기
  ```

## 2. 기록할 내용 정리
- 방금 대화에서 그 파트에 해당하는 **확정 항목들**을 뽑는다. 각 항목은 카드 필드 5개로 정리:
  - **결정**: 무엇을 결정했는지 한 줄
  - **값**: 수치·조건(여러 개면 `- ` 불릿으로 쪼갠다)
  - **근거**: 저자·연도·Table/쪽 또는 계산 출처
  - **적용**: 어디에 쓰는지
  - **파일**: 산출물/스크립트/refs 경로
- 그리고 색인용 **짧은 요약** 2개: `결정 요약`(≤15자 권장), `핵심 값`(가장 중요한 수치 1개).
- **날짜**는 `MM-DD`(오늘 날짜). **상태 아이콘**을 정한다(확정=✅, PDF직독=📄, 미확정=⚠️, 폐기=🗑, 대기=⏳).

## 3. ID 정하기 (덮어쓰기/추가 판정)
- `## 상세 기록`의 기존 카드들에서 같은 항목(같은 대상의 결정)을 찾는다.
  - **있으면 → 그 카드의 `dNN`을 재사용**(덮어쓰기).
  - **없으면 → 새 ID**: 기존 최대 `dNN` + 1 (없으면 `d01`). 두 자리 0패딩.

## 4. 색인 행 반영 (덮어쓰기/추가)
- `## 색인` 표에서 같은 항목 행(같은 `[▶](#dNN)` 링크)을 찾는다.
  - 있으면 → 그 줄을 **통째로 덮어쓴다**: `| MM-DD | <단위공정> | <결정 요약> | <핵심 값> | <상태> | [▶](#dNN) |`
  - 없으면 → 표 끝(또는 날짜순 적절한 위치)에 **새 줄 추가**.

## 5. 상세 카드 반영 (덮어쓰기/추가)
- `## 상세 기록`에서 `<a id="dNN">` 카드를 찾는다.
  - 있으면 → 그 카드(제목~마지막 불릿)를 **통째로 덮어쓴다**. 옛 내용은 남기지 않는다.
  - 없으면 → `## 상세 기록` 끝에 **새 카드 추가**:
    ```
    ### <a id="dNN"></a>MM-DD · <단위공정> — <결정 요약> <상태아이콘>
    - **결정**: ...
    - **값**: ...
    - **근거**: ...
    - **적용**: ...
    - **파일**: ...
    ```
- 어떤 결정이 **이전 항목을 정정·대체**하면, 옛 카드에 `➡️ **후속**: [#dNN](#dNN)에서 정정` 한 줄을 남기고 옛 카드 상태를 🗑/⚠️로 바꾼다(이력 보존).
- 다른 카드·다른 파트는 건드리지 않는다.

## 6. 프로젝트 전용 — 글로벌은 절대 건드리지 않음
- 이 명령은 **이 프로젝트의 `연구노트.md`만** 갱신한다. 글로벌 마스터(`~/.claude/연구_라이브러리/마스터_연구노트.md`)는 **절대 자동 반영하지 않는다**.
- 이유: 글로벌은 모든 프로젝트가 매 세션 읽는 공용 색인이라, 주제가 다른 프로젝트의 값까지 섞이면 오염된다.
- 글로벌에 올릴 만한 **재사용 가능한 문헌 지식**(저자·Table 근거 + 프로젝트 바꿔도 쓸모 있는 표준값/전환율/방법)이 있으면, 사용자가 명시적으로 **`/글로벌업데이트`** 를 호출할 때만 반영한다.

## 7. 요약 보고
- 무엇을 **덮어썼고**(옛값→새값) 무엇을 **새로 추가**했는지(색인 행 + 카드 짝) 표로 짧게 보고한다. 파일 경로(연구노트.md)도 함께.
- **ALL 모드면** 표에 **변동 없어 건너뛴 파트**도 같이 보여준다(갱신 / 추가 / 건너뜀 구분). → 무엇이 안 바뀌었는지도 한눈에 확인 가능.
- 글로벌에 올릴 만한 재사용 지식이 보이면, 보고 끝에 "글로벌 후보: …(원하면 `/글로벌업데이트`)" 한 줄만 제안한다(자동 반영 금지).
