대상: **$ARGUMENTS**

`/주제분석`으로 수집·랭킹을 끝낸 뒤 실행하는 **본 분석 단계**다. 아래 "분석 한방" 파이프라인을 수행한다. 결과는 한국어, 표 중심. **추측 금지**(확인 안 된 값은 `확인 필요`), 모든 수치·판정엔 근거(저자·연도·Table/쪽·검색 출처) 병기.

> 목적: 수집한 논문들을 정독해 **극복할 가치가 있는 한계점**을 추려내고, 거기서 **점수화된 노벨티**를 뽑아 다음 단계(`/가상시뮬레이션`)에 넘긴다.
> 참고: `refs`의 **필독 📖**은 *당신이 직접 출력해 읽는 배경지식용*이라 분석 대상 선정과 무관하다. 이 단계는 **수집한 논문 전체**를 분석한다.

## 0. 대상 확정
- `refs/00_문헌목록.md`의 **전체 논문 목록**을 분석 대상으로 삼는다($ARGUMENTS에 번호가 있으면 그 번호만).
- 각 논문의 확보 경로 판정: `refs/NN_*.pdf`가 있으면 **PDF 정독**, 없으면 **web에서 원문/출판본 확보**(전문 불가 시 초록+본문 일부). 둘 다 안 되면 `접근불가`로 표시.

## 1. 정독 — 서브에이전트 병렬
- **논문 1편당 `정독분석가` 하나씩** 병렬로 띄운다(한 메시지에 여러 Task 호출). 각 팀원에게 자기 논문의 **refs 번호·경로**와 주제/탐색 기준을 넘긴다.
- 편수가 많으면(예: ~50편) **배치로 나눠** 모두 돌린다(한 배치 끝나면 다음 배치). 중간에 누락 없이 전체를 커버할 것.
- 각 팀원은 방법·핵심결과·**한계점**(근거 위치 + 극복여부 검색 + 개선난이도 + 확보 깊이[정독/web-부분/접근불가])을 돌려준다.
- 대량 정독·검색을 팀에 떼어 **메인 대화는 깨끗이** 유지.

## 2. 추합·분류 (메인)
팀 결과의 한계점을 **병합**하고 분류한다:
- **공통 한계점** — 여러 논문에서 반복되는 것(어느 논문들인지 번호 병기). 노벨티 1순위 후보.
- **개선 가능** — 극복 경로가 보이는 한계.
- **개선 불가** — 근본적 제약(물리·열역학 한계 등). 명시하되 노벨티 출발점에서는 제외.
- **이미 극복된 한계점은 출력에서 제외** — 정독분석가가 `극복됨`으로 판정한 것(어떤 문헌이 극복했는지 한 줄로 기록만 남기고 본 분류표에는 넣지 않는다).
- 접근불가로 정독 못 한 논문은 별도로 표시(분석 한계 명시).

## 3. 노벨티 도출·점수화 — 서브에이전트 병렬
- 분류된 한계점(**공통·개선가능 우선**)을 묶어 `노벨티설계가`를 병렬로 띄운다. 한계가 많으면 주제/공정 단계별로 나눠 여러 명에게 배분.
- **노벨티 방향을 `/주제분석`에서 정한 주제 유형에 맞춘다**:
  · **A(선행연구 풍부)** → 기존 SOTA 대비 **명시적 차별화**(새 촉매·조건·통합·원료)가 노벨티의 핵심.
  · **B(신규 조합)** → **직렬 연결(통합) 자체**가 노벨티 + 링크별 개선. 한 논문의 출력 스트림을 다음 논문 입력으로 잇는 공정 체인이 성립하는지 확인.
  · **C(기존제품·신규방법)** → **신규 방법(kinetic·신규촉매 등)의 적용**이 노벨티. 방법 논문의 조건을 외부 계산(MATLAB 등)으로 검증해야 할 수 있음을 명시.
- 각 팀원은 노벨티 + **3축 점수**(독창성=선행연구 부재 / 구현 가능성 / 임팩트)와 **종합점수**(기본 가중치 0.4/0.3/0.3, $ARGUMENTS에 가중치 있으면 그 값)를 돌려준다.
- 메인은 노벨티를 **종합점수 내림차순**으로 통합한다.

## 4. 종합 출력 (대화창)
- **한계점 분류표**: 분류(공통/개선가능/개선불가) | 한계점 | 관련 논문(번호) | 극복여부 | 개선난이도. (극복된 것 제외)
- **노벨티 점수표**(종합점수 내림차순): 노벨티 | 출발 한계(번호) | 독창성(점) | 구현(점) | 임팩트(점) | 종합점수 | 선행연구 유무(출처)
- **추천 1~3**: 상위 노벨티 + "왜 유망한지" + **가상 시뮬레이션으로 검증할 핵심 가설** 한 줄.

## 5. 엑셀 정리
- 한계점·노벨티를 JSON으로 만들어: `python "${CLAUDE_PLUGIN_ROOT}/scripts/make_novelty_excel.py" <입력JSON> <출력xlsx>`
- 출력: `data/results/분석시작_<주제요약>.xlsx` (한계점 / 노벨티 2시트).
- 파일이 열려 있어 저장 실패 시 `_v2` 새 이름으로 저장 후 알림.

## 6. 마무리
- 확정 한계·노벨티(채택할 것)는 **`/업데이트`**(인자없음=바뀐 파트 전체 자동 갱신)로 연구노트(`연구노트.md`)에 기록한다(특정 파트만 콕 집으려면 `/업데이트 노벨티`처럼. 같은 항목 덮어쓰기·새 항목 추가, **프로젝트 `연구노트.md`만** — 글로벌 마스터는 건드리지 않는다. 재사용 가능한 문헌 지식은 `/글로벌업데이트`로만 반영).
- **파트 맵 씨앗 심기**: 채택한 연구방향의 **공정 체인을 단위공정 목록으로 펴서** 연구노트의 `## 파트 맵 (공정 체인)`에 기록한다(없으면 만든다). 예: `단계1 바이오매스→에탄올 : 전처리·가수분해·당화·분리·재순환` / `단계2 에탄올→SAF : 탈수·분리·올리고머·수소화·증류`. → 이 목록이 `/업데이트 all`의 파트 후보가 된다(이후 `/가상시뮬레이션` 블록 분해가 더 정밀하게 갱신).
- **다음 단계 안내**: 상위 노벨티를 연구방향(대안)으로 잡아 **`/가상시뮬레이션 <대안 번호 또는 노벨티>`** 로 물리적 타당성·수렴을 검증하라고 안내한다.
