새 연구 프로젝트(현재 폴더)를 PSE_demo1 파이프라인에 맞게 세팅한다. 결과는 한국어. **기존 파일은 함부로 덮어쓰지 않는다**(아래 규칙).

> 목적: 플러그인을 다른 컴퓨터에 설치한 직후, 프로젝트 루트에 **읽을 수 있는 `README.md`·`CLAUDE.md`**를 깔고 폴더 골격을 만든다. (플러그인 자체는 커맨드·에이전트·스킬만 제공하고 프로젝트 파일은 안 만들기 때문.)

## 0. 현재 폴더 확인
- 작업 대상은 **현재 폴더(프로젝트 루트)**다. 경로를 한 줄로 보여준다.
- `$ARGUMENTS`에 "덮어쓰기"가 있으면 기존 파일도 새 템플릿으로 교체한다. 없으면 **이미 있는 파일은 건드리지 않고 건너뛴다**(사용자 작업물 보호).

## 1. CLAUDE.md 깔기
- 루트에 `CLAUDE.md`가 **없으면** 플러그인 템플릿을 복사한다:
  `cp "${CLAUDE_PLUGIN_ROOT}/templates/CLAUDE.md" "./CLAUDE.md"`
- **있으면** 기본은 건너뛴다. ("덮어쓰기" 모드면 교체하되, 교체 전 한 줄로 알린다.)
- 깔았으면 안내: 파일 안 `<...>` 부분(주제·목표·현재 단계)을 채우라고 한 줄로 알린다.

## 2. README.md 깔기
- 루트에 `README.md`가 **없으면** 플러그인 템플릿(파이프라인 사용 가이드)을 복사한다:
  `cp "${CLAUDE_PLUGIN_ROOT}/templates/README.md" "./README.md"`
  (이 템플릿은 명령 6종·서브에이전트·핵심 습관·폴더 규칙을 담은 **프로젝트용 사용 가이드**다. 개발자용 플러그인 README가 아니다.)
- **있으면** 건너뛴다("덮어쓰기" 모드 제외). 사용자가 이미 프로젝트 README를 쓰고 있을 수 있으니 보호한다.

## 3. 권장 폴더 골격 생성
- 다음 폴더를 없을 때만 만든다(`mkdir -p`로 안전하게):
  `data/raw`, `data/results`, `refs`, `scripts`, `sections`
- 각 폴더 용도는 CLAUDE.md "폴더 규칙"과 동일. 새로 만든 폴더만 보고에 표시한다.

## 4. 글로벌 연구노트(선택)
- `~/.claude/연구_라이브러리/마스터_연구노트.md`는 **프로젝트 자동 기록 대상이 아니다**. `/업데이트`는 프로젝트 `연구노트.md`만 갱신하고, 글로벌은 사용자가 **`/글로벌업데이트`** 를 호출할 때만 기록된다(파일 없으면 그때 자동 생성).
- 미리 만들고 싶으면 한 줄로 물어본다(필수 아님). "예"면 `templates/마스터_연구노트.md`를 그 경로로 복사한다(폴더 없으면 생성).

## 5. 요약 보고
- 표로 짧게: 항목 | 경로 | 결과(새로 깔림 / 이미 있어 건너뜀 / 덮어씀). CLAUDE.md·README.md·폴더 5개 모두 포함.
- **다음 단계 안내**: CLAUDE.md의 `<...>`를 채운 뒤 **`/주제분석 <주제>`**로 파이프라인을 시작하라고 한 줄로 안내.
