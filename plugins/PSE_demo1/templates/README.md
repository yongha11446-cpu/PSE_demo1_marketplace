# 연구 프로젝트 — 사용 가이드

> PSE_demo1 플러그인 제공 템플릿. `/초기화` 시 프로젝트 루트에 복사된다.
> 빈 폴더에서 **논문탐색 → 가상 시뮬레이션 → 아스펜 모델링 → 경제성·환경성 평가**까지
> 한 연구를 처음부터 재현하도록 돕는 워크플로우다.
> 핵심은 6개 슬래시 명령 + 3종 서브에이전트 + 연구노트(기록 시스템)의 조합.

---

## 0. 핵심 철학 3가지 (모든 명령이 이 철학을 따름)

자세한 규칙은 [CLAUDE.md](CLAUDE.md) 참고.

1. **검증** — md를 믿지 말고 원본 PDF를 직접 읽어 대조한다.
2. **발췌** — 모든 수치엔 (저자, 연도, Table/Figure, 쪽수) 근거가 붙는다.
3. **기록·갱신** — 바뀌면 즉시 연구노트를 갱신한다 (anti-stale). 옛 값은 남기지 않는다.

## 1. 명령어 6종 — 연구 흐름 순서대로

| 단계 | 명령 | 언제 쓰나 | 무엇을 하나 |
|---|---|---|---|
| ① 세팅 | `/초기화` | 새 연구 프로젝트 시작할 때 | 템플릿 CLAUDE.md·README + 폴더 골격 생성 |
| ② 주제분석 | `/주제분석` | 연구 주제를 막 정했을 때 | 주제 유형 분류·논문탐색·노벨티·연구방향 3대안·엑셀을 **한 방에** |
| ③ 심층분석 | `/분석시작` | 논문을 다 모았을 때 | 전체 정독 → 한계점 분류(공통/개선가능/개선불가) → 노벨티 점수화 |
| ④ 시뮬 | `/가상시뮬레이션` | 대안이 정해졌을 때 | 블록별 조건·근거발췌·수지/수렴계산·물리타당성·엑셀 |
| ⑤ 기록 | `/업데이트` | 값·결정을 확정할 때마다 | **프로젝트** 연구노트(색인+카드)에 기록 — 같은 항목 덮어쓰기 (글로벌 미반영) |
| ⑥ 글로벌 | `/글로벌업데이트` | 재사용 가능한 문헌 지식을 공유할 때 | **사용자 명시 호출 시만** 글로벌 마스터 연구노트에 반영(게이팅) |

## 2. 단계별 실전 사용법

### `/주제분석` — 주제 정한 직후 딱 한 번
```
/주제분석 <연구 주제>
```
- `논문탐색가` 서브에이전트가 병렬로 논문을 폭넓게 수집·가치평가(저널 등급·주제 적합도)하고,
  노벨티를 뽑아 **연구방향 3대안**(공정체인 형태)과 엑셀까지 만든다.
- 끝나면 **반드시 `/업데이트`** 로 확정 주제·대안을 연구노트에 기록한다.

### `/분석시작` — refs/에 PDF 다 넣은 뒤
- refs/ 안의 논문 1편당 `정독분석가` 1명이 병렬로 붙어 정독한다.
- 각 한계점이 **후속/최신 문헌에서 이미 극복됐는지**까지 검색해 판정 → 공통/개선가능/개선불가로 분류.
- 그 뒤 `노벨티설계가`가 독창성·구현가능성·임팩트 3축으로 노벨티를 점수화.
- **팁**: PDF를 refs/에 먼저 넣어야 web 원문 대신 PDF 직독이 된다(검증 품질↑).

### `/가상시뮬레이션` — 대안 확정 후
- 공정 블록을 **하나씩** 잡고, 매 블록마다 refs 발췌 → 수지/수렴 계산 → 물리타당성 검증 → 엑셀.
  (블록 구성은 연구 주제·대안마다 다름. 대안 정의에서 블록 체인을 읽어온다.
  *예시(ATJ 표준 골격):* 전처리·가수분해·발효 → 에탄올 정제 → 탈수 → 올리고머화 → 수소화 → 분리·증류)
- **무거운 계산은 scripts/의 파이썬으로** 빠지고 대화창엔 결과 표만 출력된다.
- 결과는 **정답이 아니라** 본모사 진입 전 사전점검 + 본모사와 대조할 **기준선**이다.

### `/업데이트` — 값 하나 확정할 때마다 습관처럼
```
/업데이트              ← 인자 없음: 바뀐 파트 전체 자동 갱신
/업데이트 <파트>        ← 특정 파트만 (예: 탈수촉매)
```
- `근거/연구노트.md`의 색인표 + 상세 카드를 갱신.
  같은 항목은 **덮어쓰기(최신만 유지)**, 옛 값은 남기지 않는다.
- **프로젝트 연구노트.md만** 갱신한다. 글로벌 마스터는 건드리지 않는다.

### `/글로벌업데이트` — 재사용 지식을 프로젝트 밖으로 공유할 때만
- 다른 프로젝트에서도 쓸 **문헌 지식**(근거 有 + 횡단 재사용)이 생겼을 때 **사용자가 직접** 호출한다.
- `~/.claude/연구_라이브러리/마스터_연구노트.md`에 게이팅 통과 항목만 누적(파일 없으면 자동 생성).
- 프로젝트 한정 값(투입량·조성·방침)은 올리지 않는다 — 다른 프로젝트를 오염시키기 때문.

## 3. 서브에이전트 3종 (명령이 내부에서 병렬 호출)

| 에이전트 | 역할 |
|---|---|
| `논문탐색가` | 다수 논문 폭넓게 탐색·수집, 저널 등급·주제 적합도로 가치평가, 한계점·노벨티 추출 |
| `정독분석가` | 배정받은 논문 1편 정독(PDF 우선), 방법·핵심결과·한계점 추출, 후속문헌 극복 여부 판정 |
| `노벨티설계가` | 한계점을 받아 후속연구 노벨티 도출, 독창성·구현가능성·임팩트 3축 점수화 |

## 4. 이 플러그인을 "완벽하게" 쓰는 핵심 습관 5가지

1. **PDF를 먼저 refs/에 넣어라.** 모든 명령이 "md 믿지 말고 원본 직독"으로 동작한다. PDF 없으면 web fallback이라 검증 등급이 떨어진다.
2. **명령 → 즉시 `/업데이트`.** 결과가 나오면 바로 연구노트에 기록해야 다음 세션이 "옛 스트림으로 틀린 답"을 하지 않는다.
3. **아스펜 설계는 수동.** 플러그인은 블록 근거·계산·검증만 돕고 **자동으로 설계하거나 스킬을 갱신하지 않는다**. 한 블록 설치 → 검증 통과 → 다음 블록.
4. **수치엔 항상 (저자, 연도, Table, 쪽). 단위 항상 명시.** 근거 못 찾으면 `⚠️확인 필요`. 이게 교수 보고/논문에 그대로 쓰인다.
5. **상태 아이콘으로 진척 관리**: ✅확정 · 📄PDF직독 · ⚠️확인필요 · 🗑폐기 · ⏳대기. 연구노트 색인표만 봐도 어디가 미확정인지 한눈에 보인다.

---

## 5. 폴더 규칙 (요약)

| 폴더 | 용도 |
|---|---|
| `refs/` | 논문 원문(PDF) + 요약 md |
| `data/raw/` | 입력 원자료(조성 등, 수정 금지) |
| `data/results/` | 계산 결과(xlsx/csv) |
| `scripts/` | 계산 파이썬 (무거운 계산은 여기로) |
| `sections/` | 단계별 문서 (필요한 것만 lazy loading) |
| `근거/연구노트.md` | 설계 결정 + 날짜별 변경 로그 (기록 중심) |
